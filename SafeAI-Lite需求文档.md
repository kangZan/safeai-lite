# SafeAI-Lite 数据脱敏工具需求文档（轻量版）

## 1. 引言

### 1.1 工具定位

SafeAI-Lite 是一款轻量级的数据安全辅助工具，主要面向需要使用云端AI（如Coze、DeepSeek等）进行内容创作、数据分析，但又担心企业敏感信息泄露的普通用户。它通过简单的文本识别和替换技术，帮助用户在向AI发送内容前自动隐藏敏感信息，并在获取AI结果后恢复真实内容，全程无需部署复杂的人工智能模型，所有处理均在本地完成。

### 1.2 目标用户

- 企业内非技术人员（如行政、市场、运营等），经常使用AI辅助撰写报告、整理资料。
- 对数据安全有要求的个人用户，希望在使用AI时保护隐私信息。
- 不需要复杂语义识别，仅需快速脱敏常见敏感词（如姓名、电话、地址等）的轻量场景。

### 1.3 核心价值

- **安全**：所有敏感信息在本地处理，不上传云端，杜绝泄露风险。
- **简单**：无需学习复杂规则，通过界面操作即可完成脱敏与还原。
- **兼容**：支持多种常见文件格式（Word、Excel、PDF、TXT、LOG），输出为通用的Markdown格式，方便复制粘贴到AI工具。

## 2. 功能概述

SafeAI-Lite 提供两大核心功能模块：

- **敏感实体策略配置**：让用户自定义哪些信息属于敏感信息，以及如何替换它们。
- **文案脱敏**：上传或粘贴原始内容，工具自动识别敏感信息并替换为占位符；用户拿到AI返回结果后，再通过工具还原真实内容。

另外，工具界面会显示“意图脱敏”和“代理设置”两个菜单，但当前版本暂不实现，点击时会提示“敬请期待”。

## 3. 敏感实体策略配置

### 3.1 功能描述

本模块允许用户查看和管理工具内置的敏感信息类别（称为“敏感实体”），并可添加自己的自定义实体。每个实体都可以设置是否生效以及使用哪种替换策略。所有需要复杂人工智能模型支持的替换策略（如数字比例化、数字模糊化）将被置灰，不可选择。

### 3.2 内置敏感实体

工具预置了以下常见敏感信息类别，用户无法删除，但可以调整每个实体的生效状态和替换策略：

- 物理地址（如街道、小区等）
- 邮箱地址
- 姓名 / 用户名
- 公司 / 组织名称
- 电话号码
- 银行账户 / 卡号
- URL 网址
- IP 地址
- **地名机构**（地名 + 机构类型词，如"北京某某科技有限公司"、"广东某大学"）

### 3.3 替换策略说明

在基础版本中，只支持两种简单替换策略：

| 策略名称         | 说明                                                         | 可逆性             |
| :--------------- | :----------------------------------------------------------- | :----------------- |
| **随机数据替换** | 将识别出的敏感信息替换为一个随机生成的唯一代号（例如 `[姓名_1]`、`[邮箱_2]`）。工具会记录下代号与原始值的对应关系（脱敏表），后续可以还原。 | 可逆（可还原）     |
| **置空**         | 直接将敏感信息删除或替换为空字符串。                         | 不可逆（无法还原） |

> **注意**：其他策略（如数字比例化、数字模糊化）因需要人工智能模型支持，在基础版中显示为灰色，无法选择。

### 3.4 自定义敏感实体

用户可以根据自身业务需要，添加新的敏感信息类别。例如，公司内部的项目代号“火星计划”需要脱敏。

**新增实体步骤**：

1. 点击“新增实体”按钮。
2. 填写以下信息：
   - **实体名称**：给这类敏感信息起个名字，例如“项目代号”。
   - **同义词列表**：列出这个实体的常见写法或变体，每行一个。例如“火星计划”、“火星项目”、“Mars Project”。工具在识别时会匹配这些词语。
   - **匹配正则**：填写一个正则表达式（一种文本匹配规则），用于更精确地识别这类信息。例如，项目代号可能是“PROJ-”加数字（如 PROJ-2023-001），可以写正则表达式 `PROJ-\d{4}-\d{3}`。如果不熟悉正则，可以不填，工具会优先使用同义词列表匹配。
   - **替换策略**：选择“随机数据替换”或“置空”。
   - **生效开关**：开启后，该实体才会在脱敏时被处理。
3. 保存后，该实体将出现在自定义列表中，支持编辑和删除。

## 4. 文案脱敏（基础版）

### 4.1 功能概述

用户在此模块上传或粘贴需要处理的原始内容，工具根据已生效的敏感实体策略进行脱敏，生成脱敏后的文本（以Markdown格式展示）。用户可将脱敏文本复制给AI使用。之后，用户将AI返回的结果粘贴回工具，工具根据脱敏时记录的对应关系，将占位符还原为真实内容，并支持导出为常见文档格式。

## 4.2 脱敏流程

#### 步骤一：输入原始内容

- 在左侧操作区，有两种方式输入：
  - **上传文件**：点击上传区域，选择本地的文件（支持格式：Word文档 .doc/.docx、Excel表格 .xls/.xlsx、PDF文件 .pdf、日志文件 .log、文本文件 .txt）。工具会自动读取文件内容并显示在左侧的文本编辑框中。
  - **粘贴文本**：直接将文字粘贴到左侧的文本编辑框中。
- 左侧文本支持手动修改，方便用户微调。

#### 步骤二：识别敏感词

- 点击"识别敏感词"按钮，工具扫描文本，将所有匹配到的敏感信息汇总为**识别清单**，展示在右侧。
- 清单每一行展示：原始值、所属实体名称、替换策略（可修改）、是否排除（可勾选）。
- 用户可在清单中：
  - 修改单条词语的替换策略。
  - 勾选"排除"，让某条词语不参与脱敏。
  - 手动补充漏识别的词语（通过"添加"功能）。

#### 步骤三：执行脱敏

- 确认清单后，点击"执行脱敏"，工具按清单内容对原文进行替换：
  - 如果是"随机数据替换"，则生成唯一占位符（如 `[姓名_1]`、`[邮箱_2]`），并记录占位符与原始值的对应关系（称为"脱敏表"）。
  - 如果是"置空"，则直接删除该信息（替换为空）。
- 脱敏后的完整文本会显示在右侧的"脱敏结果"区域，并自动转换为Markdown格式。用户可直接点击"复制"按钮，将脱敏文本粘贴到任何AI聊天工具中。
- 同时，用户可以点击"导出为MD"按钮，将脱敏结果保存为 `.md` 文件，方便后续使用。

#### 步骤三：还原AI返回的内容

- 当AI根据脱敏后的文本生成回复后，用户将AI返回的内容（可以是文字，也可以是包含占位符的文本）粘贴到下方“还原区”的文本框中。
- 点击“还原”按钮，工具会读取本次脱敏时记录的脱敏表，将文本中出现的占位符（如 `[姓名_1]`）替换回真实的原始值。
- 还原后的内容会显示在还原结果区域。
- 用户可以选择“导出”还原结果，支持的格式有：
  - Word文档（.docx）
  - Excel表格（.xlsx）——如果还原内容是表格结构，导出为Excel可保持行列。
  - PDF文件（.pdf）
  - 纯文本文件（.txt）
- 选择格式后，点击导出，即可保存还原后的真实文档。

### 4.3 会话与脱敏表管理

- 每次脱敏操作都会自动生成一个会话，以时间命名（如“2025-03-14 10:30”），显示在右侧的会话列表中。
- 每个会话中保存了本次脱敏的原始内容、脱敏后内容、脱敏表（占位符与原始值的映射）。用户可以在会话列表中切换查看历史记录，方便多次还原或复查。
- 用户也可以手动删除不需要的会话，彻底清除敏感信息映射，确保数据安全。

## 6. 非功能需求

### 6.1 安全性

- 所有文件读取、文本处理、脱敏表存储均在用户本地计算机完成，绝不联网上传任何原始内容。
- 脱敏表与会话绑定，以加密形式存储在本地，用户可随时清理历史会话。
- 无任何用户行为追踪或数据收集。

### 6.2 易用性

- 界面简洁，按钮功能一目了然，无需专业知识即可操作。
- 提供示例说明和悬浮提示，帮助用户理解同义词列表和正则表达式的填写方法。

### 6.3 性能

- 处理速度：对于常见大小的文档（如10页以内的Word或Excel），脱敏时间不超过3秒。
- 支持的文件大小上限建议为20MB，过大文件可提示用户分段处理。

### 6.4 兼容性

- 支持主流操作系统：Windows 10/11、macOS 12+、Ubuntu 20.04+（若为桌面应用）。
- 若为Web应用，需支持Chrome、Edge、Safari等现代浏览器。

## 7. 数据存储说明

工具在本地创建以下数据：

- **配置文件**：保存用户对内置实体策略的修改、自定义实体列表、界面偏好等。
- **会话数据库**：记录每次脱敏的原始内容、脱敏后文本、脱敏表。用户可手动清理。

所有数据均存储于用户指定的目录（如文档/SafeAI-Lite），不写入系统敏感区域。

## 8. 附录：常见问题

**Q：什么是“脱敏表”？**
A：脱敏表是工具记录的一个对照清单，例如：`[姓名_1] = 张三`，`[邮箱_2] = zhangsan@company.com`。脱敏时生成，还原时使用。

**Q：我不懂正则表达式，可以正常使用吗？**
A：可以。自定义实体时，您可以只填写“同义词列表”，工具会直接匹配这些词语。正则表达式是更高级的匹配方式，不是必填项。

**Q：脱敏后的文本发给AI，AI能理解吗？**
A：可以。占位符（如`[姓名_1]`）保留了上下文的关联性，AI通常能理解它们代表特定人物或事物，不影响分析效果。例如：“请总结一下[姓名_1]的报告”，AI会知道需要总结某人的报告。

**Q：为什么有些策略是灰色的？**
A：基础版聚焦于简单、快速的文本替换，不依赖人工智能模型。灰色策略（如数字比例化）需要更复杂的语义理解，将在未来高级版本中提供。
